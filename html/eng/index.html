(<code>0b0BBBBBBB</code>) is an ASCII string.</p> </li> <li> <p>Recursive case: if <code>s</code> is an ASCII string, and <code>c</code> is an ASCII character, then <code>sc</code> is an ASCII string.</p> </li> </ul> </div> <div class="paragraph"> <p>It can be readily verified that each ASCII string is a binary string. See Bootstrap for a complete proof.</p> </div> </div> </div> <div class="sect1"> <h2 id="_tokens">Tokens</h2> <div class="sectionbody"> <div class="paragraph"> <p>The key building blocks of the syntax include reserved symbols. In order to encode these symbols for users, we introduce special syntax involving quotes and prefixes.</p> </div> <div class="sect2"> <h3 id="_literals">Literals</h3> <div class="paragraph"> <p>A <strong>single quoted literal string</strong> is defined recursively.</p> </div> <div class="ulist"> <ul> <li> <p>Base case: the following are single quoted literal strings:</p> <div class="ulist"> <ul> <li> <p>(in other words, all characters <strong>except</strong> <code>'</code>).</p> </li> <li> <p>(<code>\w</code>).</p> </li> <li> <p><code>\'</code></p> </li> </ul> </div> </li> <li> <p>Recursive step: if <code>l</code> is a single quoted string and <code>c</code> is a binary string satisfying the base case, then <code>l.c</code> is a single quoted string.</p> </li> </ul> </div> <div class="paragraph"> <p>A <strong>double quoted literal string</strong> is defined recursively.</p> </div> <div class="ulist"> <ul> <li> <p>Base case: the following are single quoted literal strings:</p> <div class="ulist"> <ul> <li> <p>(In other words, all characters <strong>except</strong> <code>"</code>).</p> </li> <li> <p>(<code>\w</code>).</p> </li> <li> <p><code>\"</code></p> </li> </ul> </div> </li> <li> <p>Recursive step: if <code>l</code> is a double quoted string and <code>c</code> is a binary string satisfying the base case, then <code>l.c</code> is a double quoted string.</p> </li> </ul> </div> <div class="paragraph"> <p>A <strong>literal string</strong> is a literal single quoted string or a literal double quoted string.</p> </div> </div> <div class="sect2"> <h3 id="_number_systems">Number Systems</h3> <div class="paragraph"> <p>As with above, binary strings start with <code>"0b"</code>. This subset can be <strong>completely</strong> described by a regular expression, or in a simple recursive statement. This is called the Minimal Grammar.</p> </div> <div class="paragraph"> <p>In fact, the <strong>entire</strong> semantics can be expressed with the Minimal Grammar; this is done in the Boostrap. However, for expressivity, we include the Full Grammar, allowing for arbitrary nesting of terms.</p> </div> <div class="sect2"> <h3 id="_minimal_grammar">Minimal Grammar</h3> <div class="paragraph"> <p>For simplicity, we exclude most of the concepts defined in Tokens.</p> </div> <div class="paragraph"> <p>The <strong>Minimal Welkin</strong> language is defined as follows:</p> </div> <div class="ulist"> <ul> <li> <p>A part is either <code>[a-z]</code> or <code>[A-Z]</code>.</p> </li> <li> <p>An arrow is a sequence <code>(p1 "--" p2? "-&#8594;" p3)</code>, where <code>p1</code>, <code>p2</code>, and <code>p3</code> are parts.</p> </li> <li> <p>A web is a sequence of either parts or arrows, enclosed by brackets (<code>"{"</code> and <code>"}"</code>). This sequence may be empty. In this case, the web is denoted by <code>{}</code>.</p> </li> <li> <p>A term is either a part, a web, or an arc.</p> </li> <li> <p>A "Minimal Welkin" string is defined recursively:</p> <div class="ulist"> <ul> <li> <p>Base case: any term.</p> </li> <li> <p>Recursive case: if <code>s</code> is an accepted string and <code>t</code> is a term, then <code>s,t</code> is accepted by the grammar.</p> </li> </ul> </div> </li> </ul> </div> </div> <div class="sect2"> <h3 id="_full_grammar">Full Grammar</h3> <div class="paragraph"> <p>Refer to Tokens for more details on literals and number systems.</p> </div> <div class="paragraph"> <p>` start -&#8594; terms `</p> </div> <div class="paragraph"> <p>It can be readily shown that the full grammar is compatible with the Minimal Grammar. Additionally, this grammar is <strong>LL(0)</strong>, ensuring it is unambiguous. Refer to the Bootstrap for complete proofs.</p> </div> </div> </div> </div>
